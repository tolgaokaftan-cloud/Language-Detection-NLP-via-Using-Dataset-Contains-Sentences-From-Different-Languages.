Language Detection NLP via Using Dataset Contains Sentences From Different Languages.

 

Business Problem and Statement:



In this project, We are developing the end-to-end NLP pipeline for language detection. Our main objective of the system is to recognize the language of the "given text".

The thing in general is that the textual informations may come from different countries and regions, individuals, customers, or platforms. On our world we need to figure it out while we have some or customer reviews, messages, comments, some answers to surveys, and feedback about products that may be in different languages. That is prior to applying any other techniques like sentiment analysis, topic modeling, translation, or chatbot automation, the organization must understand the language in which the text was written.

Thus, language detection becomes an integral part of a greater NLP system. With the automatic language detection, the organization can route the message to the appropriate department, use the correct language model, increase the quality of customer support, and understand its international users better.

The problem statement is formulated as the supervised multi-class text classification problem in my project. The input to the model will be a text sentence or paragraph, and the output is the language prediction.

Business Problem shortly:

The business problem is that the firm gets text data in a variety of languages, and manually recognizing the language of the text is time consuming and repetitive.
In case when there is an increase in text data, it will be hard for the people to go through every text manually.

The development of an automated language detection model will help in processing text data in multiple languages more efficiently and effectively. There are some ways how it can benefit the business:

* Automatically organizing customer messages by language
* Sending customer support requests to the exact language team
* Improving multilingual chatbot systems
* Supporting international market analysis
* Reducing manual workload and with that it leads time saving.
 

Data Collection:



In a real business environment, this type of data could be collected from different text sources such as customer reviews, emails, journals, external sources like statista, support tickets, social media comments, product feedback forms, or website messages.

For our project, I found and got a prepared dataset that includes texts and their corresponding language labels. Since the dataset already contains labeled examples, it is ready to apply for supervised machine learning.
 

NLP Formulation for Our Case:



This is a language detection task. Therefore it is a type of text classification problem.

The system takes a text as input and predicts its language as output.

* Example:

Input text:
Wie geht es dir?

Expected output:
German

* Even we need to see the right guess whenever we write a word for:

Input text:
Erdbeere

Expected Output:
German

So, the model learns the relationship between textual patterns and language labels. After training, it should be able to predict the language of new unseen text samples.

Preprocessing
 
In this stage, we preprocess our text data before training our NLP model. Preprocessing of texts is crucial in any NLP pipeline since raw text data may include unnecessary white spaces, inconsistencies in letter cases, or even missing values.

However, in case of this particular language detection task, I have to avoid applying a very aggressive form of preprocessing. The reason behind it is that language detection is based on certain words, letters, accents, and other character combinations. After we eliminate too many characters, we may lose some valuable information.

Special characters like é, ü, ñ, ç, or ğ, we have to be very helpful in recognizing various languages. Therefore, we will go for simple preprocessing techniques.

The main preprocessing steps are:

* Removing rows with missing target labels.
* Converting text values into string format.
* Converting text to lowercase.
* Removing extra spaces.
* Removing empty text rows whether there are any.
* Preparing the input variable and target variable for model training

def clean_text(text):

    text = re.sub(r"\s+", " ", text)
    text = text.strip()

    return text
 
A text cleaning function we wrote above. Firstly, the text is transformed into string data type to guarantee that all values are correctly converted. After that, I transform the text into lowercase. In this way, the machine will consider words written with different capital letters as one pattern.

Punctuation marks, accents, and special language signs are not removed since they may help identify the language.
 
We imported the "re" library to use regular expressions during text preprocessing. We use that to remove extra spaces from the text. This makes the text cleaner before model training.

Feature Extraction with TF-IDF

 
Here, we transform the cleaned data into numeric features so then we can apply the Machine Learning algorithms which are not capable to understand the raw text data, this is why the text data needs to be converted to a numeric form before feeding it to the machine learning algorithm.

In this case, we perform a transformation process using TF-IDF which stands for Term Frequency-Inverse Document Frequency. This emphasizes on the terms or patterns of characters which help to distinguish the texts.

As this project involves language detection, I perform character-level TF-IDF vectorization, not just the word-level TF-IDF vectorization. This is due to the fact that languages can be detected from the character patterns.
In [50]:
tfidf_vectorizer = TfidfVectorizer(
    analyzer="char",
    ngram_range=(2, 5),
    max_features=50000
)
 
In this step, we use "TfidfVectorizer" to transform text data into features.

The reason why we chose "analyzer=char"` is that this model is a language detection problem. In the case of language detection problems, the character-based patterns are very helpful. Each language might contain different letters, accents, or characters such as "ç", "ğ", "ñ", "é" or patterns such as "sch". These can be important for distinguishing one language from another.

For the "ngram_range", we set "ngram_range=(2, 5)" because we try to the vectorizer to get the character combinations that consist of 2 to 5 characters, at least we can start by those numbers. The main point here is that the model will not only analyze the single character letters but also character combinations. These character combinations might show the writing pattern of the languages.

For the "max_features" parameter, we chose "max_features=50000". Because we do not want to let the vectorizer create as many features as it wants. If there is no restriction for the number of features, the vectorizer can create a lot of unnecessary features. It makes the model heavier and less effective.
In [51]:
X_train_tfidf = tfidf_vectorizer.fit_transform(X_train)
X_test_tfidf = tfidf_vectorizer.transform(X_test)
 
Fitting is done only on the training data. This is because the test data is supposed to simulate unseen data.

1. fit_transform() is called for training data since the vectorizer has to be trained with the patterns in training data and then transform those patterns to numeric features.

2. transform() is called for test data since no learning has to take place on the test data.

Model Evaluation From the Results Below



This section is about evaluating the performance of the baseline model.

It is necessary to perform model evaluation after its training. It will help to understand the model's behavior on unseen data and make an appropriate conclusion about the model's reliability regarding the business problem.

In this case, the following metrics are used for model evaluation:

* Accuracy
* Precision
* Recall
* F1-score
* Classification report
* Confusion matrix

Accuracy shows the percentage of correct predictions. However, in order to take into account that this is a multi-classification problem, I will look at precision, recall, and F1-score for each class of languages.

The base model reached the accuracy score of about 0.97 which implies that 97% of language labels were predicted by the model.

It seems to be a great result since it is evident from my experience that the combination of character-level TF-IDF features and the Multinomial Naive Bayes model is effective for the language detection.

As I analyze the classification report, I can note that almost all languages have very high precision, recall and F1-score values. There are several languages like Arabic, Dutch, Indonesian, Korean, Spanish, Swedish, and Tamil which reached perfect or close to perfect results.

The value of the macro average F1-score is 0.97 and the value of the weighted average F1-score is also 0.97. Thus, it is evident that the model demonstrates good performance for the language classification. The similarity of the macro and the weighted averages indicates that the balance of the model performance for language classes is rather high.

Still, there are some weaknesses. For instance, the English class has the precision score of 0.65 but the recall score equals 1.00. It implies that the model correctly recognized all real English samples but made incorrect predictions regarding non-English samples.

The language Chinese has a recall value of 0.84, implying that some Chinese documents were misclassified into another class of language. Similarly, the languages Portuguese and Pushto also have slightly low recall values than the best-performing classes.

In general, the classifier has excellent results; however, according to the classification report, there is some scope of improvement with regard to some of the languages. The next phase involves analyzing the confusion matrix and the incorrectly predicted language classes.

Comparison of Models and Final Model Selection



In the current project, the problem statement is that a company should automatically recognize the language of given text. It is necessary since a company receives customer queries, comments on products, feedbacks on social networks and emails in a variety of languages and therefore, automatic recognition of the language will help to direct customer messages in a right direction, use an appropriate NLP model.

The problem is formulated as a supervised multi-class NLP classification problem, where input is the preprocessed text and output is the predicted language label.

The first step in building the system was loading the dataset and exploring it. After that, the text data was preprocessed and transformed to numerical features by means of character-level TF-IDF since languages are usually differ from each other with the usage of different letters, accent marks, etc.

The modeling step involved training of two models – Multinomial Naive Bayes and Logistic Regression, where the former one is a baseline model and the latter one is used as a second alternative.

From the result of evaluation, it can be noted that Logistic Regression was better. Naive Bayes was 0.9722 and by Logistic Regression was 0.9770. Moreover, Logistic Regression was also more accurate in the calculation of macro and weighted F1-scores.

As Logistic Regression was the best algorithm for language detection, it was selected as the classifier for the task of language detection.

Generally, the model is strong enough because of high accuracy, interpretability, and the possibility to detect several languages simultaneously. However, the biggest disadvantage of the model is that there is still confusion between some languages, particularly in cases of very short texts and similar languages' alphabet. This classifier should be used as the initial step of the multilingual NLP pipeline in business.

Final Discussion

 
In this project, I built an NLP pipeline for language identification. My pipeline works with multilingual text data that may be customer reviews, emails, tickets, and comments on social media platforms. After loading the dataset, I have done data exploration to know about the distribution of languages, null values, duplicates, and lengths of texts.
